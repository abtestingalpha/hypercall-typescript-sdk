# Security Policy

## API and Protocol Issues

If you discover a vulnerability in Hypercall protocol, exchange, contracts, or backend behavior, report it through the
private security channel used by the Hypercall team.

Do not open public GitHub issues for vulnerabilities that could affect funds, trading behavior, authentication,
signatures, or user data.

## SDK Issues

If you find a security problem in this SDK, report it privately through GitHub security advisories after the repository
is public, or through the private Hypercall team channel before handoff.

Include:

- Affected SDK version or commit.
- Reproduction steps.
- Expected and actual behavior.
- Whether keys, signatures, balances, orders, withdrawals, or user data are affected.
